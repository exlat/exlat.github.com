Introduction to ExLat Project

LaTeX, being a popular typesetting tool, it would be useful if there is some software that interchange various document formats and LaTeX. This includes both export to latex and import from latex. Potential formats are html, xml, different Google document formats(ICS format of Google calendar), etc.

ExLat is free software released under GNU GPL License

Use Cases

The following use cases are identified.


Google Calendar

As an initial step we are concentrating on exporting Google calendar (ICS format) to LaTeX Table. All the events related to a particular "Summary string" is converted to a Latex table.

Image Slides

Suppose we have a sequence of slides in some image format, say .jpg. These slides can be converted to latex-beamer slides.

Research Publication Portals

These days the most preferred format for Researchers to submit their publications is Latex. There are web portals like http://www.ijcaonline.org, www.springerlink.com which maintains details about various papers which collects data from these Latex files. Now the process is done manually. Automation of this would benefit the research community.

HTML to Latex

Many a times we need to embed a webpage in latex document. A HTML to Latex converter helps here. The main challenge is to process HTML and CSS documents to render it correctly in Latex.

User Interface

It is desirable to give an option for the user to choose his own latex template. This way the user has the flexibility to choose his own customized theme. The dynamic content produced is inserted at predefined positions in the template file. The user should mark these positions with special keywords. The approach is similar to ruby on rail's web template. This is illustrated in the following example.

\documentclass[12pt]{article} \begin{document} %user_data \end{document}

Here %user_data is the key word which is replaced by the dynamic content.
